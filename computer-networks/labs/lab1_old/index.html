The lab consists in writing a TCP client-server application. The server implements a service for its clients (i.e. sends the list of  files in a given directory, or the current time on the server’s machine, etc). The server runs until explicitly shutdown. Only one connection is handled at any moment for this example.

 

Your problem implementation will typically handle multiple clients at a time !

 

Server command line call

./<server> <LISTEN_PORT>

 

The server bind its listening socket on all addresses of the local machine and on the port specified in the command line.

 

The client connects to the server, issues a request, gets back the answer and displays it on the screen.

Client command line call:

./<client> <SRV_IP_ADDR> <SRV_PORT>

 

The client connect to <SRV_IP_ADDR>:<SRV_PORT> in order to contact the server.

 

Useful calls (see man pages):

  1. socket
  2. htonl,  htons,  ntohl,  ntohs - convert values between host and network byte order
  3. bind
  4. connect
  5. listen
  6. accept
  7. read, write, recv, send
  8. close, shutdown

 

Notice:

Lab 1 is due in week 3 (of the semester).

Each student will code a different service for the server. Similar implementations won’t be accepted.

You can use  any available technique for handling concurrent clients on the same server. (processes-fork, multiple threads… etc)
